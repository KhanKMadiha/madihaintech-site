## Featured Project: AI Support Chatbot

This project documents the design and development of a conversational AI chatbot built as part of my ongoing learning.

The goal is to demonstrate how I approach conversational AI using a support engineering mindset: clear user guidance, structured decision paths, and thoughtful handling of ambiguity.

### Background

After starting with IBM Watson and then Google Dialogflow CX (which hit persistent permission and domain allowlisting issues), I migrated to the Anthropic API. The key is secured via a Cloudflare Worker proxy, with CORS locked to the portfolio domain and zero client-side exposure. The chatbot auto-opens on page load, maintains session history, and runs consistently across all pages.

### What This Project Demonstrates

- **Systems thinking** - designing scalable conversation paths
- **Security architecture** - Cloudflare Worker proxy pattern, no secrets in client-side code
- **User intent handling** - anticipating needs and routing appropriately
- **Applied experience** - translating real-world support patterns into AI systems
